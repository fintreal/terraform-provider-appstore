# terraform-provider-appstore

Terraform provider for managing Apple App Store Connect resources (bundle identifiers, provisioning profiles, certificates).

- Repo: `git@github.com:fintreal/terraform-provider-appstore.git`
- Default branch: `main`
- Terraform Registry: `fintreal/appstore`

## Tech Stack

- Go 1.25
- Terraform Plugin SDK v2
- App Store Connect API client, vendored in-repo at `internal/appstore/` (formerly the standalone `fintreal/app-store-sdk-go`, now folded in; package `openapi`)
- GoReleaser for builds/releases
- GPG-signed releases

## Project Structure

```
main.go                          # Entry point, serves the provider plugin
provider/
  provider.go                    # Provider definition, auth config, resource registry
  bundleidentifier/              # appstore_bundle_identifier resource
    schema.go
    operations/{create,read,delete}.go
  provisioningprofile/           # appstore_provisioning_profile resource
    schema.go
    operations/{create,read,delete}.go
  certificate/                   # appstore_certificate data source (read-only)
    schema.go
    operations/read.go
internal/appstore/               # Vendored App Store Connect client (package openapi)
  test/                          # Client live integration tests (need KEY_DATA/KEY_ID/ISSUER_ID)
examples/                        # Example Terraform configs (used by tfplugindocs)
docs/                            # Auto-generated provider documentation
.github/
  workflows/release.yml          # CI: verify gate (build/vet/unit tests), semver, docs, goreleaser
  workflows/test.yml             # CI: unit tests, folded SDK integration tests, terraform apply/destroy
  test/main.tf                   # Integration test Terraform config
```

The folded integration tests read `KEY_DATA`/`KEY_ID`/`ISSUER_ID`; CI maps these from the existing `APPSTORE_KEY`/`APPSTORE_KEY_ID`/`APPSTORE_KEY_ISSUER_ID` secrets (same App Store Connect key). They also rely on one fixed distribution certificate (`64A2UF5HD4`) existing in the account.

## Key Commands

```sh
go install .                     # Build and install the provider binary
go mod tidy                      # Tidy dependencies (GOPRIVATE=github.com/fintreal/*)
```

For local development, use a `~/.terraformrc` with `dev_overrides`:
```hcl
provider_installation {
  dev_overrides {
    "fintreal/appstore" = "$HOME/go/bin"
  }
}
```

## Provider Auth

Requires three credentials (via provider config or env vars):
- `APPSTORE_KEY` - App Store Connect API private key
- `APPSTORE_KEY_ID` - API key ID
- `APPSTORE_KEY_ISSUER_ID` - Issuer ID

## Conventions

- Resource CRUD operations live in `provider/<resource>/operations/` as separate files
- Schema definitions live in `provider/<resource>/schema.go`
- Docs are auto-generated by `tfplugindocs` from `examples/` and schema descriptions; do not edit `docs/` manually
- Release is automated: push to `main` runs a verify gate (`go build`/`go vet`/`gofmt`), then semver bump, doc generation (pinned `tfplugindocs`), and goreleaser
- Commit messages follow conventional commits (e.g., `feat:`, `fix:`, `chore:`)
- Private Go modules: set `GOPRIVATE=github.com/fintreal/*`
